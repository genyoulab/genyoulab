# 연차 카운터 애니메이션 오류 수정 보고서 (report_FIX06_카운터수정.md)

## 1. 개요
- **작업 목표**: `brand.html` 페이지에서 스크롤로 카운터 섹션 뷰포트 진입 시 "0 Years"에서 카운트업 되지 않고 멈춰 있는 현상 수정
- **수정 방향**: `setInterval` 대신 `requestAnimationFrame`을 사용하고, `ease-out` 이징 기법을 적용하여 1500ms(1.5초) 동안 0에서 18까지 부드럽게 증가하는 애니메이션 로직으로 전면 교체

## 2. 작업 내역
- **대상 파일**: `/brand.html`
- **적용 로직 (JavaScript)**:
  - 기존의 일정한 시간 간격으로 숫자를 올리는 불안정한 `setInterval` 방식을 제거했습니다.
  - 디스플레이의 주사율에 맞춰 부드럽게 렌더링되는 `requestAnimationFrame` 기반의 `animateCounter` 함수를 새롭게 정의했습니다.
  - `progress` 비율을 구한 후, `1 - Math.pow(1 - progress, 3)` 수식을 통해 애니메이션 후반부로 갈수록 서서히 느려지는 고급스러운 `ease-out` 효과를 구현했습니다.
  - 기존 구조대로 `id="counter-section"`(또는 `id="year-counter"`) 요소가 화면에 진입하는 시점(`isIntersecting === true`)을 `IntersectionObserver`로 감지합니다.
  - 애니메이션 1회 실행 후 `observer.disconnect()`를 호출하여 불필요한 관찰이나 애니메이션 중복 발생을 완전히 차단했습니다.

## 3. 검증
- **[완료]** `brand.html` 페이지 스크롤 후 카운터 요소가 노출되었을 때 숫자가 정상적으로 0에서 18까지 올라가는지 확인했습니다.
- **[완료]** 애니메이션의 시작과 끝부분 진행 속도가 `ease-out` 곡선에 맞게 부드럽게 이뤄지는 것을 확인했습니다.
- **[완료]** 뷰포트 밖으로 벗어났다가 다시 스크롤 다운해도 카운터가 재실행되지 않도록(1회만 실행) 최적화 상태를 검수했습니다.
