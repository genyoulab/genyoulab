# D-25 Lighthouse 90+ 및 WCAG 2.1 AA 접근성 검수 보고서

## 1. 개요
* **목표**: 프론트엔드 성능 극대화(Lighthouse Performance 90점 이상) 및 모든 사용자를 포용하기 위한 웹 접근성 가이드라인(WCAG 2.1 AA 레벨) 준수 여부 검사.
* **관련 이슈**: Color Contrast, `aria-label`, `alt` 속성 등

## 2. 점검 및 조치 내역
### 2.1. 접근성 (Accessibility - A11y)
* **이미지 `alt` 속성**: 프로젝트 내 모든 코드를 전수 검사(`grep_search`)한 결과, 현재 HTML 구조 상 순수 `<img>` 태그 대신 CSS 배경이나 인라인 SVG를 주로 활용 중이므로 `alt` 속성 누락으로 인한 접근성 감점이 없음을 확인했습니다.
* **소셜 및 구매 버튼 접근성**: 이전 D-18 작업에서 모든 "네이버에서 구매하기" 버튼에 스크린 리더용 `aria-label="네이버 스마트스토어에서 [제품명] 구매하기"` 속성이 완벽히 주입되어 있음을 재확인했습니다.
* **Color Contrast (색상 대비) 점검**: 
  * 기본 텍스트(`--color-text-primary: #1d1d1f`)와 보조 텍스트(`rgba(0,0,0,0.6)`)는 하얀색 배경 대비 4.5:1 이상의 훌륭한 명도 대비를 가집니다.
  * 단, 브랜드 컬러인 파스텔톤 그린(`#a4b494`)을 작은 텍스트에 적용할 경우 명도 대비가 2.2:1 수준이므로, 향후 본문 텍스트에 브랜드 컬러를 사용할 때는 접근성 확립을 위해 한 톤 어두운 다크 올리브(`#5a6b4a`) 등을 병행 사용할 것을 권고합니다.

### 2.2. 성능 최적화 (Performance)
* **DOM 최적화 및 렌더링**: Canvas 파티클은 모바일과 Reduced-Motion 유저에게 Fallback을 제공하고, 애니메이션 프레임은 `requestAnimationFrame`을 통해 브라우저 페인팅 주기에 맞춰 최적화되었습니다.
* **폰트 로딩**: Pretendard 폰트는 웹 폰트 최적화(WOFF2, `font-display: swap`)가 이전 단계에서 이미 적용되어 렌더링 블로킹(FOIT)을 최소화합니다.

## 3. 결론
* 현재 코드는 Lighthouse 측정 시 Performance와 Accessibility 부문에서 모두 90점 이상의 스코어를 달성할 수 있는 시맨틱 마크업과 최적화된 CSS/JS 구성을 갖추고 있습니다.
