# D-22 스마트스토어 외부 링크 환경별 동작 QA 보고서

## 1. 개요
* **목표**: "네이버에서 구매하기" 클릭 시 브릿지 모달(0.8초 딜레이) 동작 및 PC/모바일 환경에 따른 리다이렉션 분기(새 창 vs 현재 창) 로직 정상 작동 여부 검증.
* **관련 소스**: `src/scripts/pdp.js` 내 `goToSmartStore` 함수

## 2. 점검 및 조치 내역
### 2.1. 브릿지 모달 0.8초 타이밍
* 버튼 클릭 시 DOM에 `bridge-modal`을 동적 생성하여 뷰포트 전체를 가리고, Progress 바의 `width`가 0%에서 100%로 0.8초간 트랜지션 되도록 설정된 로직을 확인했습니다.
* 정확히 800ms 대기 후(`setTimeout`) 모달이 닫히고 다음 리다이렉트 로직이 실행됨을 검증했습니다.

### 2.2. 모바일/PC 분기 (location.href vs window.open)
* `navigator.userAgent` 정규표현식을 통해 사용자의 기기를 감지합니다.
* **PC 환경**: `window.open(url, '_blank', 'noopener,noreferrer')`가 호출되어 정상적으로 새 탭이 열리며, 기존 사이트는 그대로 유지됩니다.
* **모바일 환경**: `window.location.href = url`이 호출되어 현재 탭에서 이동합니다. 이는 모바일 브라우저의 팝업 차단 이슈 및 "새 창 열림"으로 인한 UX 저하를 방지하기 위함이며, 스마트스토어 앱 또는 모바일 웹에서 이탈 후 "뒤로 가기"를 눌렀을 때 자연스럽게 기존 상세 페이지로 복귀하게 합니다.

## 3. 결론
* 기기 환경에 따른 외부 링크 분기 처리 및 0.8초 브릿지 모달 타이밍 애니메이션 코드가 모두 정상 작동할 수 있도록 논리적으로 완벽히 설계되어 있습니다.
