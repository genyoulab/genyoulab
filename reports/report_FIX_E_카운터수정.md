# 수정 보고서: 브랜드 페이지 연차 카운터 수정

## 1. 개요
* **목표:** `brand.html`의 브랜드 연차 카운터가 "0"에서 멈춰있는 현상을 수정하고, 스크롤 진입 시 "0"에서 "18"까지 1.5초 동안 카운트업 되는 애니메이션을 적용합니다.
* **작업 대상 파일:** `brand.html`

## 2. 상세 작업 내역

### 2.1 카운터 요소 ID 변경
기존 HTML에 `id="year-counter"`로 지정되어 있던 부분을 요구사항에 맞게 `id="counter-years"`로 변경했습니다.
* **수정 전:** `<div class="counter-num" id="year-counter">0</div>`
* **수정 후:** `<div class="counter-num" id="counter-years">0</div>`

### 2.2 카운트업 스크립트 추가
`brand.html`의 최하단(`</body>` 직전)에 Intersection Observer 기반의 카운트업 자바스크립트를 추가했습니다.
* **애니메이션 방식:** `requestAnimationFrame`과 시간 기반의 진행률 계산을 통해 1.5초(1500ms) 동안 부드럽게(ease-out cubic) 숫자가 증가하도록 구현했습니다.
* **스크롤 트리거:** `IntersectionObserver`를 활용하여 카운터 요소가 뷰포트에 50% 이상 진입(`threshold: 0.5`)했을 때 애니메이션이 시작되도록 설정했습니다.
* **중복 방지:** 스크롤을 여러 번 위아래로 움직이더라도 애니메이션은 1회만 동작(`observer.disconnect()`, `animated` 플래그 사용)하도록 제어했습니다.
* **최종값 고정:** 연산 오차를 방지하기 위해 애니메이션 종료 시 정확히 '18'이라는 숫자로 고정되도록 강제했습니다.

## 3. 검증 (Verification)
* `brand.html` 로드 시 하단으로 스크롤하여 카운터 영역이 보일 때 애니메이션이 즉시 발동하는 코드가 올바르게 삽입되었습니다.
* ID 매칭이 정확하여 스크립트가 타겟 요소를 정상적으로 탐색합니다.

로컬 브라우저에서 새로고침 후 `brand.html`의 해당 영역으로 스크롤하여 카운트업 애니메이션이 예상대로 동작하는지 확인해 주시기 바랍니다.
